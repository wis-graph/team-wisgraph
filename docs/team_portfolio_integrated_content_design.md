## 웹사이트 콘텐츠 + 디자인 통합 기획안

---

## 0. 기획 방향

이 웹사이트는 단순한 팀 소개 페이지가 아니다.  
목표는 “AI 강사들이 모인 팀”이 아니라 **업무 지식을 구조화하고, AI Agent와 자동화 워크플로우로 실제 시스템을 구축하는 전문 실행팀**으로 보이게 만드는 것이다.

핵심 메시지는 다음이다.

> **업무 지식을 AI가 실행할 수 있는 시스템으로.**

이 문장은 팀의 정체성을 가장 짧게 설명한다.

팀의 핵심 기반은 다음 문장으로 설명한다.

> **팀의 핵심 기반은 수학적 구조화와 컴퓨터과학적 시스템 설계입니다. 복잡한 업무를 개념·관계·규칙·프로세스로 분해하고, 이를 AI Agent와 자동화 워크플로우가 활용할 수 있는 구조로 구현합니다.**

웹사이트 전체는 다음 순서로 설계한다.

1. 첫 화면에서 시스템 구축팀이라는 인상을 준다.
2. Clients & References 로고 영역으로 신뢰를 빠르게 만든다.
3. Capabilities에서 실제 구축 역량을 보여준다.
4. System Architecture에서 “우리가 만드는 시스템”을 시각화한다.
5. Method에서 수학적 구조화와 CS 기반 설계 철학을 보여준다.
6. Client Work에서 실제 적용 가능한 업무 영역을 보여준다.
7. Education & Enablement에서 초보자부터 임원까지 교육 가능함을 보여준다.
8. Team of Specialists에서 멤버는 짧고 역할 중심으로만 소개한다.
9. Engagements에서 의뢰 가능한 프로젝트 형태를 제시한다.
10. Contact에서 상담으로 연결한다.

# 5. System Architecture

## 목적

“이 팀이 만드는 시스템이 무엇인지”를 시각적으로 보여준다.

이 섹션은 매우 중요하다.  
단순 교육팀이나 자동화 툴 강사처럼 보이지 않으려면, **시스템 아키텍처를 보여주는 섹션**이 반드시 필요하다.

---

## 섹션명

**System Architecture**

한국어 병기:

**업무 지식이 AI 실행 시스템이 되는 구조**

---

## 섹션 카피

흩어진 문서, 규정, 노하우, 반복 업무를 하나의 업무지식 구조로 정리하고,  
AI Agent가 이를 검색·판단·작성·실행할 수 있도록 시스템화합니다.

---

## 다이어그램 구조

```text
[Input Layer]
문서 / 규정 / 매뉴얼 / 회의록 / 상담기록 / 업무 노하우

        ↓

[Ontology Layer]
개념 / 관계 / 규칙 / 예외 / 프로세스 / 역할

        ↓

[Agent Layer]
검색 Agent / 보고서 Agent / 고객응대 Agent / 콘텐츠 Agent / 분석 Agent

        ↓

[Workflow Layer]
Make / n8n / Dify / API / Claude Code / 내부 DB

        ↓

[Operation Layer]
보고 / 응답 / 기록 / 발행 / 알림 / 승인 / 관리
```

---

## 각 레이어 설명

### Input Layer

조직 안에 흩어진 문서, 규정, 매뉴얼, 회의록, 상담기록, 업무 노하우를 수집합니다.

### Ontology Layer

업무 개념, 관계, 규칙, 예외, 프로세스를 구조화해 AI가 참조할 수 있는 지식 체계를 만듭니다.

### Agent Layer

업무 역할별 Agent를 설계합니다. Agent는 단순 답변을 넘어 검색, 판단, 작성, 실행을 지원합니다.

### Workflow Layer

Agent의 결과를 실제 업무 시스템과 연결합니다. Make, n8n, Dify, API, Claude Code 등을 활용합니다.

### Operation Layer

보고서 작성, 고객 응대, 콘텐츠 발행, 회의록 정리, 알림, 승인 등 실제 업무 실행으로 연결합니다.

---

## 디자인 디렉션

- 전체를 5개 수평 레이어로 표현
- 각 레이어를 얇은 선과 작은 노드로 연결
- 마우스 hover 시 각 레이어 설명 노출
- 색상은 절제
- 강조는 Ontology Layer와 Agent Layer에 집중
- 배경은 어두운 그리드 또는 subtle noise

## 본문

AI 자동화의 핵심은 도구 연결이 아니라 업무 지식의 구조화입니다.  
우리는 복잡한 업무를 개념·관계·규칙·예외·프로세스로 분해하고,  
이를 AI Agent와 자동화 워크플로우가 활용할 수 있는 실행 구조로 구현합니다.

팀의 핵심 기반은 수학적 구조화와 컴퓨터과학적 시스템 설계입니다.  
복잡한 업무를 감으로 자동화하지 않고, 먼저 구조화하고 모델링한 뒤,  
Agent와 Workflow가 실제 업무에서 작동할 수 있도록 설계합니다.

---

## 6단계 프로세스

### 01. Observe

업무를 관찰합니다.

- 고객 인터뷰
- 현장 분석
- 기존 문서 확인
- 반복 업무 탐색
- 병목과 리스크 파악

### 02. Decompose

업무를 작은 단위로 분해합니다.

- 업무 개념 추출
- 작업 단위 분해
- 역할과 책임 구분
- 입력/출력 정의
- 예외 상황 정리

### 03. Model

업무 지식을 모델링합니다.

- 개념 관계 정의
- 규칙·예외 구조화
- 프로세스 맵 작성
- 데이터 흐름 정리
- 온톨로지 설계

### 04. Architect

Agent와 자동화 구조를 설계합니다.

- Agent 역할 정의
- 시스템 아키텍처 설계
- API/도구 연동 방식 결정
- Human-in-the-loop 설계
- 파일럿 범위 설정

### 05. Build

실제 시스템을 구축합니다.

- Agent 구현
- 자동화 워크플로우 구축
- 내부 업무도구 개발
- 테스트 및 개선
- 운영 가이드 작성

### 06. Enable

조직 안에 정착시킵니다.

- 수준별 교육
- 실습 워크숍
- 운영 매뉴얼
- 후속 코칭
- 현장 피드백 반영

---

## 디자인 디렉션

- 6단계 타임라인
- 각 단계는 숫자 + 동사 + 짧은 설명
- 스크롤하면서 단계가 하나씩 나타나는 구성
- 배경에는 얇은 선형 프로세스 그래픽
- 너무 컬러풀하게 만들지 않기

# 13. 피해야 할 느낌

## 피해야 할 인상

- 바이브코딩으로 급하게 만든 랜딩페이지

- 멤버 이력서 모음
- 밝고 가벼운 SaaS 템플릿
- “누구나 쉽게 AI Agent” 같은 저가형 메시지

## 피해야 할 문장

- 누구나 쉽게 AI를 만들 수 있습니다.
- 10분 만에 Agent 완성
- 노코드로 빠르게 자동화하세요.
- AI 강의 전문팀
- ChatGPT 활용법을 알려드립니다.

## 대신 쓸 문장

- 업무 지식을 AI가 실행할 수 있는 시스템으로 구현합니다.
- 업무를 자동화하기 전에, 먼저 업무 지식을 구조화합니다.
- 복잡한 업무를 개념·관계·규칙·프로세스로 분해합니다.
- AI Agent와 자동화 워크플로우가 활용 가능한 구조를 설계합니다.
- 조직의 수준에 맞게 교육하고, 실제 업무에 정착시킵니다.

## Capabilities

## System Architecture

**System Architecture**

흩어진 문서, 규정, 노하우, 반복 업무를 하나의 업무지식 구조로 정리하고,  
AI Agent가 이를 검색·판단·작성·실행할 수 있도록 시스템화합니다.

Input Layer  
문서 / 규정 / 매뉴얼 / 회의록 / 상담기록 / 업무 노하우

Ontology Layer  
개념 / 관계 / 규칙 / 예외 / 프로세스 / 역할

Agent Layer  
검색 Agent / 보고서 Agent / 고객응대 Agent / 콘텐츠 Agent / 분석 Agent

Workflow Layer  
Make / n8n / Dify / API / Claude Code / 내부 DB

Operation Layer  
보고 / 응답 / 기록 / 발행 / 알림 / 승인 / 관리

## Method

**업무를 자동화하기 전에, 먼저 업무 지식을 구조화합니다.**

AI 자동화의 핵심은 도구 연결이 아니라 업무 지식의 구조화입니다.  
우리는 복잡한 업무를 개념·관계·규칙·예외·프로세스로 분해하고,  
이를 AI Agent와 자동화 워크플로우가 활용할 수 있는 실행 구조로 구현합니다.

Observe → Decompose → Model → Architect → Build → Enable
